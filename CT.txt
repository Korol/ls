3 Добавить таблицу“Статистика по клиенткам”: по аналогии с общая з/п таблица,
только вместо сотрудников выводим клиенток, также само отмечаем “-”, если клиентка не закреплена за сайтом,
данные вносятся вручную.
Данные подтягиваем автоматически, если клиентка закреплена за сайтом - в данную клетку
(пересечение сайт - клиентка) можно вводить текст вручную (фин статистика у них будет).

Реализация таблицы:
Fixed Cols Example: http://adazzle.github.io/react-data-grid/examples.html#/fixed-cols (фиксированные шапка и первый столбец)
Column Events Example: http://adazzle.github.io/react-data-grid/examples.html#/column-events (редактирование ячейки)


3.1. Получаем данные:
    3.1.1. Список клиентов + для каждого клиента список сайтов, с которыми связан этот клиент
    3.1.2. Список сайтов.
    3.1.3. Список финансовой статистики по связи «клиент <-> сайт» (это немного позже)

3.2. Создаём React-компонент (компонент, редъюсер, экшн) с таблицей, в которой будут отображены данные (пока без статистики):
    3.2.1. На пересечении «клиент <-> сайт»: прочерк, если клиент не закреплен за сайтом – и пустая ячейка, если закреплён.

3.3. Добавляем в компонент возможность редактирования данных в ячейках таблицы (обработчик редактирования, сохранение данных в БД):
    3.3.1. По клику на ячейку появляется возможность ввода/редактирования текста в ячейке.
    3.3.2. Результаты редактирования сохраняем по событию (onkeyup? onchange?).
    3.3.3. Для обработки события в ячейке нужен метод.
    3.3.4. Сохранение идет через dispatch и в action.

3.4. Данные сохраняем в отдельную таблицу вида: id | customerID | siteID | data :
    3.4.1. Метод сохранения в модели выполняет update (по связке customerID | siteID), а если результат update = false – тогда insert.

3.5. Добавляем в п.3.1 получение и отображение данных из новой таблицы:
    3.5.1. Полученные данные привязываем к списку клиентов (3.1.1) по схеме:
        $customer['ID']['sites'] = array(
            'siteID 1' => 'data 1',
            'siteID 2' => 'data 2',
            ...
            'siteID N' => 'data N',
        )
    3.5.2. В п.3.2.1 на пересечении вместо пустой ячейки – отображаем данные.
